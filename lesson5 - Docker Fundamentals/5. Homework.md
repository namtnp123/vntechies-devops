## Exercise 1: Containerize a Simple Web Application
## Docker Container Management Exercise

This exercise demonstrates basic Docker container lifecycle management using the traefik/whoami image. The tasks include:

### Objectives
- Deploy multiple instances of the same service
- Configure port mapping for external access
- Extract and save container logs
- Properly clean up resources

### Key Docker Commands Used
- `docker run` - Create and start containers with port mapping
- `docker logs` - Extract container logs for debugging/monitoring
- `docker stop` - Gracefully stop running containers
- `docker rm` - Remove stopped containers to free up resources

### Learning Outcomes
- Understanding container naming and port exposure
- Log management and troubleshooting techniques
- Proper container cleanup procedures
- Working with lightweight web service containers
- From image: https://hub.docker.com/r/traefik/whoami
- Run 2 whoaimi container name: `svc-1` and `svc-2` 
- Expose to host on port `8080` and `8081`
- Get log from 2 container and write to `svc-1.txt` and `svc-2.txt`
- Stop container `svc-1` and  `svc-2`
- remove container `svc-1` and  `svc-2`

## Exercise 2: Run Wordpress application as container

# Docker WordPress and MySQL Deployment Documentation

## Overview
This homework assignment involves deploying a WordPress application with MySQL database using Docker containers, ensuring data persistence across container lifecycle operations.

## Requirements
- Deploy WordPress container from official Docker Hub image
- Deploy MySQL container from official Docker Hub image  
- Ensure both containers run properly and can communicate
- Implement data persistence to retain data when containers are stopped or removed

## Implementation Notes
- Use Docker volumes or bind mounts for data persistence
- Configure proper networking between WordPress and MySQL containers
- Set up appropriate environment variables for database connection
- Test container restart scenarios to verify data persistence
- From image:
   - wordpress: https://hub.docker.com/_/wordpress
   - mysql: https://hub.docker.com/_/mysql
- Run wordpress and mysql container. make sure wordpress and mysql run properly
- Data should be persist even container stop or removed
