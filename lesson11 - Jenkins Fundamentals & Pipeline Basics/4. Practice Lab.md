# Lesson 11: Jenkins Fundamentals & Pipeline Basics - Practice Lab

## Lab Overview
This hands-on lab will help you understand Jenkins fundamentals and basic pipeline creation. You'll explore the Jenkins environment, troubleshoot existing pipelines, and implement your own pipeline from scratch.


---

## Exercise 1: Explore the Lab Environment (30 minutes)

### 1.1 Jenkins Interface Exploration (10 minutes)
**Objective**: Familiarize yourself with Jenkins dashboard and navigation

**Steps**:
1. Access Jenkins at http://<ec2-ip>:9080
2. Login with credentials: `admin` / `admin123`
3. Explore the main dashboard:
   - Note the folder structure: `DevOpsBootcamp`
   - Identify different pipeline types in the folders
   - Check the agent status (Manage Jenkins → Nodes)

**Expected Outcome**: You should see:
- DevOpsBootcamp folder with subfolders: SampleSnippets, BasicPipelines
- Docker agent node online and available
- Clean dashboard with no failed jobs

### 1.2 Explore SampleSnippets (10 minutes)
**Objective**: Understand simple Jenkins operations

**Steps**:
1. Navigate to `DevOpsBootcamp` → `SampleSnippets`
2. Run the following jobs and observe their behavior:
   - **BasicGroovyOperation**: Demonstrates Jenkins pipeline variables, scoping, and Groovy operations
   - **GitShellCurlSnippet**: Shows git clone, shell commands, and HTTP requests using curl

**Tasks**:
- Trigger each pipeline manually
- Check console output for each build
- Note the build numbers and timestamps
- Observe workspace contents after execution

### 1.3 Explore BasicPipelines (10 minutes)
**Objective**: Understand more complex pipeline structures

**Steps**:
1. Navigate to `DevOpsBootcamp` → `BasicPipelines`
2. Examine the pipeline structure:
   - **HelloWorldPipeline**: Basic pipeline structure
   - **multiStageBasic**: Multi-stage pipeline demonstration
   - **multiStageWithParameter**: Parameterized builds
   - **CowsayFrontend**: Docker-based application pipeline
   - **CowsayBackendApi**: Backend API pipeline (incomplete)

**Tasks**:
- Run `HelloWorldPipeline` and observe the stage view
- Run `multiStageBasic`
- Run `multiStageWithParameter` with different parameters
- Review the pipeline code for each job (Configure → Pipeline script)

## Exercise 2: Troubleshoot CowsayFrontend Pipeline (20 minutes)

### 2.1 Identify the Issue
**Objective**: Practice troubleshooting skills by finding configuration problems

**Background**: The CowsayFrontend pipeline has a misconfiguration that prevents it from running successfully.

**Steps**:
1. Navigate to the `CowsayFrontend` pipeline
2. Try to run the pipeline
3. Observe the failure and examine console output
4. Review the pipeline configuration

**Hint**: The issue is related to docker registry name (the registry name `daotoanhd`, should be your registry instead, modify it so the pipeline can push the image to your registry)


## Exercise 3: Implement Cowsay API Backend Pipeline (45 minutes)

### 3.1 Understand the Requirements
**Objective**: Create a complete CI/CD pipeline for the backend API

**Background**: The `CowsayBackendApi` pipeline is incomplete and only contains placeholder echo statements. You need to implement a full pipeline that:
- Clones the correct repository
- Builds a Docker image
- Runs tests
- Pushes to registry
