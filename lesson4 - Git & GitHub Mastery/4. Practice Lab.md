# Lesson 4: Git & GitHub Mastery - Practice Lab

## Lab Overview
**Duration**: 40 minutes  
**Objective**: Master Git and GitHub fundamentals through hands-on exercises  
**Format**: Individual work with pair collaboration exercises

---

## Exercise 1: Git Basics and Repository Setup (10 minutes)

### Part A: Environment Verification and Configuration
1. **Verify Git Installation**
   ```bash
   git --version
   ```
   Expected output: Git version 2.x.x or higher

2. **Configure Git (if not done already)**
   ```bash
   git config --global user.name "Your Full Name"
   git config --global user.email "your.email@example.com"
   git config --global init.defaultBranch main
   ```

3. **Verify Configuration**
   ```bash
   git config --list --global
   ```

### Part B: Create Your First Repository
1. **Create a new directory and initialize Git**
   ```bash
   mkdir my-first-repo
   cd my-first-repo
   git init
   ```

2. **Create initial files**
   ```bash
   echo "# My First Git Repository" > README.md
   echo "This is my first Git project!" >> README.md
   echo "console.log('Hello Git!');" > hello.js
   ```

3. **Check repository status**
   ```bash
   git status
   ```

4. **Add files to staging area**
   ```bash
   git add README.md
   git add hello.js
   # Alternative: git add .
   ```

5. **Make your first commit**
   ```bash
   git commit -m "Initial commit: Add README and hello.js"
   ```

6. **View commit history**
   ```bash
   git log
   git log --oneline
   ```

## Exercise 2: Branching and Merging (10 minutes)

### Part A: Create and Work with Branches
1. **Create a new feature branch**
   ```bash
   git branch feature-greeting
   git checkout feature-greeting
   # Alternative: git checkout -b feature-greeting
   ```

2. **Verify current branch**
   ```bash
   git branch
   git status
   ```

3. **Make changes in the feature branch**
   ```
   Use Visual code to edit hello.js file
   ```
4. **Commit changes**
   ```bash
   git add hello.js
   git commit -m "Add greeting function"
   ```

### Part B: Switch Branches and Merge
1. **Switch back to main branch**
   ```bash
   git checkout main
   ```

2. **Verify the file content (should be original)**
   ```bash
   cat hello.js
   ```

3. **Merge the feature branch**
   ```bash
   git merge feature-greeting
   ```

4. **Verify the merge**
   ```bash
   cat hello.js
   git log --oneline --graph
   ```

5. **Clean up: Delete the feature branch**
   ```bash
   git branch -d feature-greeting
   ```

**✅ Checkpoint**: Your main branch should now contain the greeting function.

## Exercise 3: GitHub Integration and Collaboration (15 minutes)

### Part A: Connect to GitHub
1. **Create a new repository on GitHub**
   - Go to https://github.com
   - Click "New repository"
   - Name: `git-practice-[your-name]`
   - Keep it public
   - Don't initialize with README (we already have one)

2. **Add GitHub as remote origin**
   ```bash
   git remote add origin https://github.com/[your-username]/git-practice-[your-name].git
   ```

3. **Push to GitHub**
   ```bash
   git push -u origin main
   ```

4. **Verify on GitHub website**
   - Refresh your repository page
   - You should see your files and commit history

### Part B: Fork and Clone Exercise

1. **Clone your fork locally**
   ```bash
   cd ..  # Go back to parent directory
   git clone https://github.com/[your-username]/devops-bootcamp-git-practice.git
   cd devops-bootcamp-git-practice
   ```

2. **Create a feature branch**
   ```bash
   git checkout -b add-my-info
   ```

3. **Add your information**
   ```bash
   # You can use visual code to do this
   echo "## [Your Name]" >> contributors.md
   echo "- GitHub: @[your-username]" >> contributors.md
   echo "- Role: DevOps Student" >> contributors.md
   echo "" >> contributors.md
   ```

4. **Commit and push your changes**
   ```bash
   git add contributors.md
   git commit -m "Add my information to contributors"
   git push origin add-my-info
   ```

### Part C: Create a Pull Request
1. **Go to your repo on GitHub**
2. **Click "Compare & pull request"**
3. **Fill in the pull request details:**
   - Title: "Add [Your Name] to contributors"
   - Description: "Adding my information to the contributors list"
4. **Click "Create pull request"**

## Exercise 4: Conflict Resolution (10 minutes)

### Part A: Create a Conflict Scenario
1. **Go back to your personal repository**
   ```bash
   cd ../my-first-repo
   ```

2. **Create two branches that will conflict**
   ```bash
   # Create branch A
   git checkout -b feature-a
   echo "// Feature A implementation" > feature.js
   echo "const message = 'Hello from Feature A';" >> feature.js
   git add feature.js
   git commit -m "Add Feature A"
   
   # Switch to main and create branch B
   git checkout main
   git checkout -b feature-b
   echo "// Feature B implementation" > feature.js
   echo "const message = 'Hello from Feature B';" >> feature.js
   git add feature.js
   git commit -m "Add Feature B"
   ```

### Part B: Merge and Resolve Conflict
1. **Switch to main and merge feature-a**
   ```bash
   git checkout main
   git merge feature-a
   ```

2. **Try to merge feature-b (this will create a conflict)**
   ```bash
   git merge feature-b
   ```

3. **Check the conflict**
   ```bash
   git status
   cat feature.js
   ```

4. **Resolve the conflict by editing the file**
   ```bash
   # Edit feature.js to resolve conflict
   # Remove conflict markers and choose/combine content
   echo "// Combined features implementation" > feature.js
   echo "const messageA = 'Hello from Feature A';" >> feature.js
   echo "const messageB = 'Hello from Feature B';" >> feature.js
   echo "console.log(messageA);" >> feature.js
   echo "console.log(messageB);" >> feature.js
   ```

5. **Complete the merge**
   ```bash
   git add feature.js
   git commit -m "Merge feature-b: Resolve conflict by combining features"
   ```

6. **Clean up branches**
   ```bash
   git branch -d feature-a feature-b
   ```

**✅ Checkpoint**: You should have successfully resolved a merge conflict.

## Exercise 5: Advanced Git Operations (5 minutes)

### Part A: Stashing 
1. **Make some uncommitted changes**
   ```bash
   echo "// Work in progress" >> hello.js
   echo "const temp = 'temporary work';" >> hello.js
   ```

2. **Stash the changes**
   ```bash
   git stash
   git status  # Should be clean
   ```

3. **Create a quick fix on a new branch**
   ```bash
   git checkout -b hotfix
   echo "// Critical bug fix" >> README.md
   git add README.md
   git commit -m "Fix critical bug in README"
   ```

4. **Switch back to main and apply stash**
   ```bash
   git checkout main
   git stash pop
   git status  # Should show your uncommitted changes
   ```

5. **Cherry-pick the hotfix**
   ```bash
   git cherry-pick hotfix
   ```

6. **Clean up**
   ```bash
   git add hello.js
   git commit -m "Complete work in progress"
   git branch -d hotfix
   ```

### Part B: Viewing Differences and History
1. **View differences between commits**
   ```bash
   git log --oneline
   git diff HEAD~2 HEAD  # Compare current with 2 commits ago
   ```

2. **View file history**
   ```bash
   git log --follow hello.js
   git show HEAD~1:hello.js  # Show file content from previous commit
   ```
