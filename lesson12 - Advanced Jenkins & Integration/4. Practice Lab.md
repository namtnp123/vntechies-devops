## Exercise 3: Implement Your Own Shared Library

### 3.1 Fork and Customize Shared Library

**Objective**: Create your own shared library for reusable pipeline components

**Steps**:
1. Fork https://github.com/devopsway/devops-project-cicd to your GitHub account
2. Clone your forked repository locally
3. Examine the existing library structure
4. Add new custom functions


### 3.2 Configure Your Shared Library in Jenkins

**Steps**:
1. Go to "Manage Jenkins" → "Configure System"
2. Scroll to "Global Pipeline Libraries"
3. Click "Add" to create a new library
4. Configure your library:

**Configuration Values**:
```
Name: your-devops-library
Default Version: master
Retrieval Method: Modern SCM
  Source Code Management: Git
  Repository URL: https://github.com/YOUR_USERNAME/devops-project-cicd
  Credentials: github-token
Load implicitly: ✓
Allow default version to be overridden: ✓
Include @Library changes in job recent changes: ✓
```

### 3.3 Create Organization Folder for Your Projects

**Steps**:
1. From Jenkins dashboard, click "New Item"
2. Choose "Organization Folder"
3. Name it "YourName-Projects"
4. Configure GitHub Organization settings:

**Answer Key - Configuration**:
```
GitHub Organization:
  Owner: Your Organization Name
  Credentials: github-token
  
Behaviors:
  ☑ Discover branches
  ☑ Discover pull requests
  ☑ Discover tags
  
Project Recognizers:
  ☑ Pipeline Jenkinsfile
  
Orphaned Item Strategy:
  ☑ Discard old items
  Days to keep old items: 7
  Max # of old items to keep: 10
```
