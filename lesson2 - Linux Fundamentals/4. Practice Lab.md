## 🔬 Exercise 1: File System Navigation & Operations (25 minutes)

### Objective
Master Linux file system navigation, file operations, and search capabilities through hands-on practice.

### Tasks

#### Task 1.1: Directory Navigation Mastery (8 minutes)
1. **Check current location and explore**:
   ```bash
   pwd                          # Print working directory
   ls -la                       # List all files with details
   cd /                         # Go to root directory
   ls -la                       # Explore root directory
   ```

2. **Navigate through system directories**:
   ```bash
   cd /home                     # Go to home directory
   ls -la                       # List users
   cd ~                         # Go to your home directory
   pwd                          # Confirm location
   cd /var/log                  # Go to log directory
   ls -la | head -10            # List first 10 log files
   cd -                         # Return to previous directory
   ```

3. **Practice relative and absolute paths**:
   ```bash
   cd /usr/bin                  # Absolute path
   cd ../lib                    # Relative path (up one, then lib)
   cd ../../home/ubuntu         # Multiple relative moves
   pwd                          # Verify final location
   ```

**Verification**: You should be comfortable navigating anywhere in the file system

#### Task 1.2: File and Directory Operations (10 minutes)
1. **Create practice directory structure**:
   ```bash
   cd ~

   cd devops-practice

2. **Create and manipulate files**:

   tree

   ```

**Verification**: Directory structure is organized and files are properly created and moved

#### Task 1.3: File Content Operations (7 minutes)
1. **View file contents using different methods**:
   ```bash
   # Create a sample log file
   mkdir logs
   cat > logs/application.log << 'EOF'
    2024-01-15 10:00:01 INFO Application started
    2024-01-15 10:00:05 DEBUG Database connection established
    2024-01-15 10:01:23 WARNING High memory usage detected
    2024-01-15 10:02:45 ERROR Failed to connect to external API
    2024-01-15 10:03:12 INFO User login successful
    2024-01-15 10:04:33 ERROR Database timeout
    2024-01-15 10:05:01 INFO Application shutdown
   EOF
   
   # View file contents
   cat logs/application.log                    # Display entire file
   head -n 3 logs/application.log             # First 3 lines
   tail -n 3 logs/application.log             # Last 3 lines
   less logs/application.log                  # Page through file (q to quit)
   ```

2. **Search and filter content**:
   ```bash
   # Search for patterns
   grep "ERROR" logs/application.log          # Find error lines
   grep -n "INFO" logs/application.log        # Find info lines with line numbers
   grep -v "DEBUG" logs/application.log       # Exclude debug lines
   grep -c "ERROR" logs/application.log       # Count error occurrences
   ```

3. **File information and statistics**:
   ```bash
   wc logs/application.log                    # Word, line, character count
   wc -l logs/application.log                 # Line count only
   file logs/application.log                  # File type information
   du -h logs/application.log                 # File size
   ```

**Verification**: You can efficiently view, search, and analyze file contents

## 🔐 Exercise 2: Permissions & Security (20 minutes)

### Objective
Understand and implement Linux file permissions and security best practices.

### Tasks

#### Task 2.1: Understanding Current Permissions (5 minutes)
1. **Examine existing file permissions**:
   ```bash
   cd ~/devops-practice
   ls -la                                     # View all permissions
   ls -la projects/                           # View directory permissions
   ls -la scripts/                            # View script permissions
   ```

2. **Analyze permission format**:
   ```bash
   # Create a file to analyze
   touch security-test.txt
   ls -la security-test.txt
   # Note: -rw-rw-r-- means:
   # - = regular file
   # rw- = owner can read/write
   # rw- = group can read/write  
   # r-- = others can only read
   ```

**Verification**: You understand the permission format (rwx for owner, group, others)

#### Task 2.2: Modifying Permissions (8 minutes)
1. **Practice numeric permission changes**:
   ```bash
   # Set different permission levels
   chmod 755 scripts/backup.sh               # rwxr-xr-x (executable script)
   chmod 644 docs/README.md                  # rw-r--r-- (readable document)
   chmod 600 docs/welcome.txt                # rw------- (private file)
   chmod 700 logs/                           # rwx------ (private directory)
   
   # Verify changes
   ls -la scripts/backup.sh
   ls -la docs/
   ls -ld logs/                              # -d shows directory itself
   ```

2. **Practice symbolic permission changes**:
   ```bash
   # Add execute permission for user
   chmod u+x scripts/deployment.sh
   
   # Remove write permission for group and others
   chmod go-w docs/welcome-backup.txt
   
   # Set specific permissions for different groups
   chmod u=rw,g=r,o= security-test.txt      # rw-r-----
   
   # Verify changes
   ls -la scripts/deployment.sh
   ls -la docs/welcome-backup.txt
   ls -la security-test.txt
   ```

## ⚙️ Exercise 3: Process & System Monitoring (25 minutes)

### Objective
Monitor system processes, analyze performance, and manage running applications.

### Tasks

#### Task 3.1: Process Discovery and Analysis (10 minutes)
1. **Explore running processes**:
   ```bash
   # View all processes
   ps aux                                     # All processes, detailed
   ps aux | head -10                          # First 10 processes
   ps -ef                                     # Alternative format
   
   # Find specific processes
   ps aux | grep ssh                          # Find SSH processes
   ps aux | grep ubuntu                       # Find processes by user
   pgrep ssh                                  # Get PID of SSH processes
   ```

2. **Real-time process monitoring**:
   ```bash
   # Launch real-time monitors (run each in separate terminal or use Ctrl+C to exit)
   top                                        # Basic process monitor
   # Press 'q' to quit top
   
   htop                                       # Enhanced process monitor (if available)
   # Press 'q' to quit htop
   
   # View process tree
   pstree                                     # Show process relationships
   pstree -p                                  # Include PIDs
   ```

3. **Process information gathering**:
   ```bash
   # Get detailed process information
   ps -p $$                                   # Current shell process
   ps -p $$ -o pid,ppid,cmd,etime            # Specific columns
   
   # Check process resource usage
   ps aux --sort=-%cpu | head -5              # Top CPU users
   ps aux --sort=-%mem | head -5              # Top memory users
   ```

**Verification**: You can identify and analyze running processes effectively

#### Task 3.2: Process Control and Management (8 minutes)
1. **Create and manage background processes**:
   ```bash
   # Start long-running processes
   sleep 300 &                                # 5-minute sleep in background
   ping -c 100 localhost > /dev/null &        # Background ping
   
   # Check background jobs
   jobs                                       # List background jobs
   ps aux | grep sleep                        # Find sleep process
   ps aux | grep ping                         # Find ping process
   ```

2. **Control running processes**:
   ```bash
   # Get process IDs
   SLEEP_PID=$(pgrep sleep)
   PING_PID=$(pgrep ping)
   
   echo "Sleep PID: $SLEEP_PID"
   echo "Ping PID: $PING_PID"
   
   # Terminate processes
   kill $SLEEP_PID                            # Graceful termination
   kill -9 $PING_PID                          # Force termination
   
   # Verify termination
   jobs                                       # Should show no jobs
   ps aux | grep -E "(sleep|ping)" | grep -v grep
   ```

3. **Service management basics**:
   ```bash
   # Check system services
   systemctl status ssh                       # SSH service status
   systemctl is-active ssh                    # Check if active
   systemctl is-enabled ssh                   # Check if enabled
   
   # View service logs
   sudo journalctl -u ssh --no-pager -n 10   # Last 10 SSH log entries
   ```

**Verification**: You can control processes and understand service management

#### Task 3.3: System Resource Monitoring (7 minutes)
1. **Monitor system resources**:
   ```bash
   # Memory usage
   free -h                                    # Human-readable memory info
   cat /proc/meminfo | head -5                # Detailed memory information
   
   # Disk usage
   df -h                                      # Disk space usage
   du -sh ~/devops-practice                   # Directory size
   du -sh ~/devops-practice/*                 # Size of subdirectories
   
   # CPU information
   lscpu                                      # CPU details
   cat /proc/cpuinfo | grep "model name" | head -1  # CPU model
   ```

2. **Network and system information**:
   ```bash
   # Network connections
   sudo apt install net-tools
   netstat -tuln | head -10                   # Network connections
   ss -tuln | head -10                        # Modern alternative to netstat
   
   # System information
   uname -a                                   # System information
   uptime                                     # System uptime and load
   whoami                                     # Current user
   id                                         # User and group IDs
   ```

3. **Log analysis**:
   ```bash
   # System logs
   sudo tail -n 20 /var/log/syslog           # Recent system events
   sudo grep "error" /var/log/syslog | tail -5  # Recent errors
   
   # Authentication logs
   sudo tail -n 10 /var/log/auth.log         # Recent authentication events
   
   # Analyze your application log
   cd ~/devops-practice
   grep "ERROR" logs/application.log          # Find all errors
   grep -c "INFO\|ERROR\|WARNING" logs/application.log  # Count log levels
   ```

**Verification**: You can monitor system resources and analyze logs effectively

## 🚨 Troubleshooting Guide

### Common Issues and Solutions

#### Permission Denied Errors
**Issue**: Cannot access files or directories
**Solution**: 
```bash
ls -la filename                    # Check permissions
sudo chown $USER:$USER filename    # Fix ownership
chmod 644 filename                 # Set appropriate permissions
```

#### Command Not Found
**Issue**: Commands not available
**Solution**:
```bash
which command_name                 # Check if command exists
echo $PATH                         # Check PATH variable
sudo apt install package_name     # Install missing package
```

#### Process Management Issues
**Issue**: Cannot kill processes
**Solution**:
```bash
ps aux | grep process_name         # Find correct PID
kill -9 PID                        # Force kill if necessary
sudo kill PID                      # Use sudo if needed
```

#### Package Management Problems
**Issue**: Package installation fails
**Solution**:
```bash
sudo apt update                    # Update package lists
sudo apt --fix-broken install     # Fix broken dependencies
sudo apt autoremove               # Clean up conflicts
```
