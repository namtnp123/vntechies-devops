# Practice Lab - Lesson 6: Docker Advanced & Multi-container Applications

## Lab Overview
This lab focuses on building and orchestrating multi-container applications using Docker and Docker Compose. You'll work with a full-stack application consisting of a frontend and backend service.

## Resources
- **Frontend repo**: https://github.com/devopsway/devops-project-application-cowsay-frontend
- **Backend repo**: https://github.com/devopsway/devops-project-application-cowsay-api

## Lab Tasks

### Task 1: Clone the Repositories
```bash
# Clone the frontend repository
git clone https://github.com/devopsway/devops-project-application-cowsay-frontend.git

# Clone the backend repository  
git clone https://github.com/devopsway/devops-project-application-cowsay-api.git
```

### Task 2: Build Docker Images

#### Build Backend Image
```bash
# Navigate to backend directory
cd devops-project-application-cowsay-api

# Build the Docker image
docker build -t cowsay-api:latest .

# Verify the image was created
docker images | grep cowsay-api
```

#### Build Frontend Image
```bash
# Navigate to frontend directory
cd ../devops-project-application-cowsay-frontend

# Build the Docker image
docker build -t cowsay-frontend:latest .

# Verify the image was created
docker images | grep cowsay-frontend
```

### Task 3: Start Frontend and Backend Containers

#### Start Backend Container
```bash
# Run the backend container
docker run -d \
  --name cowsay-api \
  -p 3001:3001 \
  -e NODE_ENV=production \
  -e PORT=3001 \
  cowsay-api:latest

# Verify container is running
docker ps | grep cowsay-api
```

#### Start Frontend Container
```bash
# Run the frontend container
docker run -d \
  --name cowsay-frontend \
  -p 3000:3000 \
  -e NODE_ENV=production \
  -e NEXT_PUBLIC_API_URL=http://localhost:3001 \
  -e PORT=3000 \
  cowsay-frontend:latest

# Verify container is running
docker ps | grep cowsay-frontend
```

### Task 4: Explore Container Logs

#### View Backend Logs
```bash
# View current logs
docker logs cowsay-api

# Follow logs in real-time
docker logs -f cowsay-api

# View last 50 lines
docker logs --tail 50 cowsay-api

# View logs with timestamps
docker logs -t cowsay-api
```

#### View Frontend Logs
```bash
# View current logs
docker logs cowsay-frontend

# Follow logs in real-time
docker logs -f cowsay-frontend

# View logs for troubleshooting
docker logs --tail 100 cowsay-frontend
```

#### Additional Log Commands
```bash
# View logs from both containers simultaneously
docker logs cowsay-api & docker logs cowsay-frontend

# Check container resource usage
docker stats cowsay-api cowsay-frontend

# Inspect container configuration
docker inspect cowsay-api
docker inspect cowsay-frontend
```

### Task 5: Write Docker Compose to Run Both Backend and Frontend

Create a `docker-compose.yml` file to orchestrate both services:
```yaml
version: '3.8'

services:
  cowsay-api:
    build: ./devops-project-application-cowsay-api
    image: cowsay-api:latest
    container_name: cowsay-api
    ports:
      - "3001:3001"
    environment:
      - NODE_ENV=production
      - PORT=3001
    env_file:
      - ./devops-project-application-cowsay-api/.env
    restart: unless-stopped
    networks:
      - cowsay-network

  cowsay-frontend:
    build: ./devops-project-application-cowsay-frontend
    image: cowsay-frontend:latest
    container_name: cowsay-frontend
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - NEXT_PUBLIC_API_URL=http://cowsay-api:3001
      - PORT=3000
    env_file:
      - ./devops-project-application-cowsay-frontend/.env
    depends_on:
      - cowsay-api
    restart: unless-stopped
    networks:
      - cowsay-network

networks:
  cowsay-network:
    driver: bridge
```

### Task 6: Run the Application with Docker Compose

#### Start the Multi-Container Application
```bash
# Start all services in detached mode
docker-compose up -d

# View running services
docker-compose ps

# View logs from all services
docker-compose logs

# Follow logs in real-time
docker-compose logs -f
```

#### Test the Application
```bash
# Test backend API
curl http://localhost:3001/health

# Test frontend (open in browser)
open http://localhost:3000
```

#### Manage the Application
```bash
# Stop all services
docker-compose down

# Stop and remove volumes
docker-compose down -v

# Rebuild and start services
docker-compose up --build -d

# Scale a service (if supported)
docker-compose up -d --scale cowsay-frontend=2

# View service logs individually
docker-compose logs cowsay-api
docker-compose logs cowsay-frontend
```

## Lab Verification

### Verify Setup
1. Both containers should be running: `docker-compose ps`
2. Frontend should be accessible: `http://localhost:3000`
3. Services should communicate properly within the Docker network

### Troubleshooting Commands
```bash
# Check service status
docker-compose ps

# View detailed logs
docker-compose logs --tail=100 -f

# Execute commands inside containers
docker-compose exec cowsay-api sh
docker-compose exec cowsay-frontend sh

# Check network connectivity
docker network ls
docker network inspect <network_name>

# Restart specific service
docker-compose restart cowsay-api
```

## Cleanup
```bash
# Stop and remove all containers, networks, and images
docker-compose down --rmi all --volumes --remove-orphans

# Remove unused Docker resources
docker system prune -a
```
