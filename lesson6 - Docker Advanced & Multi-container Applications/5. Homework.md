## Exercise 1: E-commerce Microservices Application (60%)

### Requirements

Build a complete e-commerce application with the following services:

#### 1. Frontend Service (React/Vue/Angular or Static HTML)
- Product listing page
- Shopping cart functionality
- Checkout interface
- Responsive design

**Technical Requirements:**
- Use multi-stage Dockerfile
- Optimize image size (< 50MB)
- Implement proper .dockerignore
- Serve via Nginx

#### 2. Product API Service (Node.js/Python/Go)
- GET /api/products - List all products
- GET /api/products/:id - Get product details
- POST /api/products - Create product (admin)
- PUT /api/products/:id - Update product (admin)
- DELETE /api/products/:id - Delete product (admin)

**Technical Requirements:**
- Multi-stage build
- Health check endpoint
- Environment-based configuration
- Proper error handling
- API documentation

#### 3. Order API Service (Node.js/Python/Go)
- POST /api/orders - Create order
- GET /api/orders/:id - Get order details
- GET /api/orders/user/:userId - Get user orders

**Technical Requirements:**
- Connect to database
- Implement health checks
- Use environment variables
- Transaction support

#### 4. Database Service (PostgreSQL/MySQL)
- Products table
- Orders table
- Users table (optional)

**Technical Requirements:**
- Data persistence with volumes
- Initialization scripts
- Health checks
- Backup strategy

#### 5. Cache Service (Redis)
- Cache product listings
- Session management (optional)

**Technical Requirements:**
- Proper volume configuration
- Connection from API services

### Docker Compose Configuration Requirements

Your `docker-compose.yml` must include:

```yaml
version: '3.8'

services:
  # All services with proper configuration
  
networks:
  # At least 2 custom networks for isolation
  
volumes:
  # Named volumes for data persistence
```

**Required Features:**
- [ ] Service dependencies (depends_on with conditions)
- [ ] Health checks for all services
- [ ] Resource limits (CPU and memory)
- [ ] Restart policies
- [ ] Environment variable configuration
- [ ] Multiple networks for security isolation
- [ ] Named volumes for data persistence
- [ ] Proper logging configuration

## Exercise 3: Production Configuration (10%)

### Requirements

Create separate configurations for development and production:

#### Development Configuration
- `docker-compose.yml` (base)
- `docker-compose.override.yml` (development overrides)

**Development Features:**
- Hot reload for frontend
- Debug mode enabled
- Volume mounts for source code
- Exposed database ports
- Verbose logging

#### Production Configuration
- `docker-compose.prod.yml`

**Production Features:**
- Optimized images only
- No source code mounts
- Restricted port exposure
- Resource limits enforced
- Production logging levels
- Health checks with proper intervals
- Secrets management (use Docker secrets or .env)

## Tips for Success

### Docker Best Practices
- Always use specific image tags, not `latest`
- Minimize layers in Dockerfiles
- Use .dockerignore to exclude unnecessary files
- Run containers as non-root users
- Scan images for vulnerabilities

### Docker Compose Best Practices
- Use environment variables for configuration
- Implement health checks for all services
- Set appropriate restart policies
- Use named volumes for data persistence
- Separate networks for security

### Development Workflow
- Test each service independently first
- Use `docker-compose logs` for debugging
- Implement proper error handling
- Write clear commit messages
- Test both development and production configurations

### Common Pitfalls to Avoid
- Hardcoding configuration values
- Not implementing health checks
- Ignoring security best practices
- Poor documentation
- Not testing data persistence
- Exposing unnecessary ports
- Not setting resource limits
