**Duration**: 40 minutes  
**Objective**: Get hands-on experience with Kubernetes cluster operations, pods, services, and configuration management.

## Exercise 1: Cluster Exploration and Pod Management (10 min)

### Objective
Explore the Kubernetes cluster and create your first pods.

### Tasks

#### 1.1 Explore the Cluster
```bash
# Check cluster information
kubectl cluster-info

# List all nodes in the cluster
kubectl get nodes

# Get detailed information about the node
kubectl describe node <node-name>

# List all namespaces
kubectl get namespaces

# View system pods
kubectl get pods -n kube-system
```

#### 1.2 Working with Namespaces
```bash
# Create a new namespace
kubectl create namespace dev-environment

# Create namespace using YAML
cat <<EOF > production-namespace.yaml
apiVersion: v1
kind: Namespace
metadata:
  name: production
  labels:
    environment: prod
    team: backend
EOF

kubectl apply -f production-namespace.yaml

# List all namespaces with labels
kubectl get namespaces --show-labels

# Get detailed information about a namespace
kubectl describe namespace dev-environment

# Set default namespace for current context
kubectl config set-context --current --namespace=dev-environment

# Verify current namespace
kubectl config view --minify | grep namespace

# Switch back to default namespace
kubectl config set-context --current --namespace=default
```

#### 1.3 Working with Resources in Different Namespaces
```bash
# Create pods in different namespaces
kubectl run nginx-default --image=nginx:1.21 --namespace=default
kubectl run nginx-dev --image=nginx:1.21 --namespace=dev-environment
kubectl run nginx-prod --image=nginx:1.21 --namespace=production

# List pods in specific namespaces
kubectl get pods --namespace=default
kubectl get pods --namespace=dev-environment
kubectl get pods --namespace=production

# List pods in all namespaces
kubectl get pods --all-namespaces

# Create a service in dev namespace
kubectl expose pod nginx-dev --port=80 --namespace=dev-environment

# List services in all namespaces
kubectl get services --all-namespaces

# Test cross-namespace communication
kubectl run test-pod --image=busybox --rm -it --restart=Never --namespace=dev-environment -- sh
# Inside the pod, try:
# nslookup nginx-dev (works - same namespace)
# nslookup nginx-dev.dev-environment.svc.cluster.local (works - FQDN)
# exit

# Delete namespace (this deletes all resources within it)
kubectl delete namespace dev-environment
```

#### 1.4 Create a Pod Imperatively
```bash
# Create an nginx pod
kubectl run nginx-pod --image=nginx:1.21

# Check pod status
kubectl get pods

# Get detailed pod information
kubectl describe pod nginx-pod

# View pod logs
kubectl logs nginx-pod

# Execute a command inside the pod
kubectl exec -it nginx-pod -- bash
```

#### 1.5 Create a Pod Using YAML
Create a file named `redis-pod.yaml`:
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: redis-pod
  labels:
    app: redis
    tier: cache
spec:
  containers:
  - name: redis
    image: redis:7
    ports:
    - containerPort: 6379
```

Apply the manifest:
```bash
# Create the pod
kubectl apply -f redis-pod.yaml

# Verify pod creation
kubectl get pods

# Check pod labels
kubectl get pods --show-labels

# Filter pods by label
kubectl get pods -l app=redis
```

#### 1.6 Pod Interaction
```bash
# Access redis CLI inside the pod
kubectl exec -it redis-pod -- redis-cli

# Inside redis-cli, test some commands:
# SET greeting "Hello from Kubernetes"
# GET greeting
# EXIT

# Delete the nginx pod
kubectl delete pod nginx-pod
```

**Expected Outcome**: Successfully create, inspect, and interact with pods.

## Exercise 3: Services and Networking (8 min)

### Objective
Expose applications using different service types and understand service discovery.

### Tasks

#### 3.1 Create a ClusterIP Service
Create `nginx-service-clusterip.yaml`:
```yaml
apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  selector:
    app: nginx
  ports:
  - protocol: TCP
    port: 80
    targetPort: 80
  type: ClusterIP
```

```bash
# Apply the service
kubectl apply -f nginx-service-clusterip.yaml

# Check service details
kubectl get services
kubectl describe service nginx-service

# Get service endpoints
kubectl get endpoints nginx-service
```

#### 3.2 Test Service from Inside Cluster
```bash
# Create a test pod
kubectl run test-pod --image=busybox --rm -it --restart=Never -- sh

# Inside the test pod, run:
# wget -qO- http://nginx-service
# exit
```

#### 3.3 Create a NodePort Service
```bash
# Delete the ClusterIP service
kubectl delete service nginx-service

# Create NodePort service
kubectl expose deployment nginx-deployment --type=NodePort --port=80 --name=nginx-nodeport

# Get service details (note the NodePort)
kubectl get service nginx-nodeport

# For minikube, get the URL
minikube service nginx-nodeport --url

# For kind, you'll need to map ports or use kubectl port-forward
kubectl port-forward service/nginx-nodeport 8080:80

# Access in browser: http://localhost:8080
```

**Expected Outcome**: Successfully create and access services.

## Exercise 4: ConfigMaps and Secrets (7 min)

### Objective
Manage application configuration using ConfigMaps and Secrets.

### Tasks

#### 4.1 Create a ConfigMap
```bash
# Create ConfigMap from literal values
kubectl create configmap app-config \
  --from-literal=APP_NAME="My Kubernetes App" \
  --from-literal=APP_VERSION="1.0.0" \
  --from-literal=LOG_LEVEL="info"

# View ConfigMap
kubectl get configmap app-config
kubectl describe configmap app-config
```

Create `app-config-file.yaml`:
```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: nginx-config
data:
  nginx.conf: |
    server {
      listen 80;
      location / {
        return 200 "Hello from ConfigMap!\n";
        add_header Content-Type text/plain;
      }
    }
```

```bash
kubectl apply -f app-config-file.yaml
```

#### 4.4 Use ConfigMap with Nginx Pod
Create `nginx-with-configmap.yaml`:
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: nginx-with-configmap
  labels:
    app: nginx-custom
spec:
  containers:
  - name: nginx
    image: nginx:1.21
    ports:
    - containerPort: 80
    volumeMounts:
    - name: nginx-config-volume
      mountPath: /etc/nginx/conf.d
  volumes:
  - name: nginx-config-volume
    configMap:
      name: nginx-config
```

```bash
# Apply the nginx pod with ConfigMap
kubectl apply -f nginx-with-configmap.yaml

# Check if pod is running
kubectl get pods nginx-with-configmap

# Test the custom nginx configuration
kubectl port-forward nginx-with-configmap 8081:80

# In another terminal, test the configuration:
# curl http://localhost:8081
# Should return "Hello from ConfigMap!"
```

#### 4.2 Create a Secret
```bash
# Create Secret from literal values
kubectl create secret generic db-secret \
  --from-literal=username=admin \
  --from-literal=password=super-secret-password

# View Secret
kubectl get secrets
kubectl describe secret db-secret

# View Secret data (base64 encoded)
kubectl get secret db-secret -o yaml
```

#### 4.3 Use ConfigMap and Secret in a Pod
Create `app-with-config.yaml`:
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: app-with-config
spec:
  containers:
  - name: app
    image: busybox
    command: ["/bin/sh", "-c", "env && sleep 3600"]
    env:
    - name: APP_NAME
      valueFrom:
        configMapKeyRef:
          name: app-config
          key: APP_NAME
    - name: DB_USERNAME
      valueFrom:
        secretKeyRef:
          name: db-secret
          key: username
    - name: DB_PASSWORD
      valueFrom:
        secretKeyRef:
          name: db-secret
          key: password
```

```bash
# Apply the pod
kubectl apply -f app-with-config.yaml

# Check environment variables
kubectl logs app-with-config | grep -E "APP_NAME|DB_"
```

**Expected Outcome**: Successfully create and use ConfigMaps and Secrets.

## Exercise 5: Troubleshooting and Debugging (5 min)

### Objective
Practice common troubleshooting techniques in Kubernetes.

### Tasks

#### 5.1 Create a Broken Pod
Create `broken-pod.yaml`:
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: broken-pod
spec:
  containers:
  - name: nginx
    image: nginx:nonexistent-tag
    ports:
    - containerPort: 80
```

```bash
# Apply the broken pod
kubectl apply -f broken-pod.yaml

# Check pod status
kubectl get pods
```

#### 5.2 Debug the Pod
```bash
# Describe the pod to see events
kubectl describe pod broken-pod

# Check for specific error messages
kubectl get events --sort-by=.metadata.creationTimestamp | grep broken-pod

# Fix the pod by editing
kubectl edit pod broken-pod
# Change image to nginx:1.21, save and exit
```

#### 5.3 Debug a Running Pod
```bash
# Create a pod with wrong command
kubectl run debug-pod --image=nginx -- /bin/sh -c "exit 1"

# Check status (should be CrashLoopBackOff)
kubectl get pods

# Check logs
kubectl logs debug-pod

# Check previous container logs
kubectl logs debug-pod --previous

# Delete the pod
kubectl delete pod debug-pod
```

#### 5.4 Common Debugging Commands
```bash
# View all events in current namespace
kubectl get events --sort-by=.metadata.creationTimestamp

# Check resource usage
kubectl top nodes
kubectl top pods

# Get all resources
kubectl get all

```

**Expected Outcome**: Successfully identify and resolve common Kubernetes issues.
