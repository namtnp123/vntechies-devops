# Lesson 7: Kubernetes Fundamentals - Lab Preparation

## Prerequisites
- Install k8s `kind`
```sh
# For AMD64 / x86_64
[ $(uname -m) = x86_64 ] && curl -Lo ./kind https://kind.sigs.k8s.io/dl/v0.30.0/kind-linux-amd64
chmod +x ./kind
sudo mv ./kind /usr/local/bin/kind
```
- Install `kubectl`
```sh
sudo apt-get update
sudo apt-get install -y kubectl
```
- Install `k9s`
```
 # Via LinuxBrew
 /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
 brew install derailed/k9s/k9s
```

- Create cluster 1 controlplane and 2 worker-node
```yaml
# config.yaml
kind: Cluster
apiVersion: kind.x-k8s.io/v1alpha4
name: devops-cluster
nodes:
- role: control-plane
  extraPortMappings:
    # API Server
    - containerPort: 6443
      hostPort: 8443
    # HTTP
    - containerPort: 80
      hostPort: 8080
      protocol: TCP
    # HTTPS
    - containerPort: 443
      hostPort: 443
      protocol: TCP
  kubeadmConfigPatches:
  - |
    kind: ClusterConfiguration
    apiServer:
      certSANs:
      - "localhost"
      - "127.0.0.1"
      - "0.0.0.0"
  - |
    kind: InitConfiguration
    nodeRegistration:
      kubeletExtraArgs:
        node-labels: "ingress-ready=true"
- role: worker
- role: worker
```
- Create cluster
```sh
kind create cluster --config=config.yaml
```
- Access cluster from local laptop
  - Copy file `~/.kube/config` to your latop
  - Edit the config file
```yaml
apiVersion: v1
clusters:
- cluster:
    certificate-authority-data: *** # <<=== Remove this
    server: https://<ec2-ip>:8443 # <<=== Change IP to EC2 IP
    insecure-skip-tls-verify: true # <<=== Add this
  name: kind-devops-cluster
contexts:
- context:
    cluster: kind-devops-cluster
    user: kind-devops-cluster
  name: kind-devops-cluster
current-context: kind-devops-cluster
kind: Config
users:
- name: kind-devops-cluster
  user:
    client-certificate-data: ***
    client-key-data: ***

```
  - Allow security group on port `8443`
  - By default, kubectl looks for a file named config in the $HOME/.kube directory. You can specify other kubeconfig files by setting the KUBECONFIG environment variable or by setting the --kubeconfig flag.
  - Test the config `kubectl get po --kubeconfig=config` or `export KUBECONFIG=$(pwd)/config && kubectl get po`

- Delete cluster
```sh
kind delete clusters devops-cluster
```
