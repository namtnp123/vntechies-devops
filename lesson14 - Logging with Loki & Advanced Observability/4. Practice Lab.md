### Exercise 2: LogQL Query Fundamentals
Practice basic LogQL syntax and filtering:

1. **Stream Selectors:**
   ```logql
   # All logs from container logs job
   {job="containerlogs"}
   
   # Logs from specific container name
   {container="log-generator"}
   
   # Logs from multiple containers
   {container=~"prometheus|grafana"}
   
   # Exclude specific containers
   {container!="cadvisor"}
   ```

2. **Line Filter Expressions:**
   ```logql
   # Filter for ERROR logs
   {container="log-generator"} |= "ERROR"
   
   # Filter for INFO logs
   {container="log-generator"} |= "INFO"
   
   # Filter for WARN logs
   {container="log-generator"} |= "WARN"
   
   # Filter out DEBUG logs
   {container="log-generator"} != "DEBUG"
   
   # Multiple filters (AND condition)
   {container="log-generator"} |= "User" |= "logged"
   ```

3. **Regular Expression Filters:**
   ```logql
   # Find logs with IP addresses
   {container="log-generator"} |~ `\d+\.\d+\.\d+\.\d+`
   
   # Find logs with specific patterns
   {container="log-generator"} |~ "User logged in.*"
   
   # Find logs with numbers
   {container="log-generator"} |~ `\d+`
   
   # Negative regex (exclude pattern)
   {container="log-generator"} !~ "DEBUG.*"
   ```

### Exercise 3: Advanced LogQL Operations
Learn advanced LogQL features for log analysis:

1. **Label Extraction:**
   ```logql
   # Extract log level using regex
   {container="log-generator"} | regexp `\[(?P<level>\w+)\]`
   
   # Extract IP addresses
   {container="log-generator"} | regexp `IP (?P<ip>\d+\.\d+\.\d+\.\d+)`
   
   # Extract request IDs
   {container="log-generator"} | regexp `request ID (?P<request_id>\d+)`
   
   # Extract memory percentage
   {container="log-generator"} | regexp `Memory usage at (?P<memory>\d+)%`
   ```

2. **Line Format Operations:**
   ```logql
   # Format output to show only timestamp and level
   {container="log-generator"} | regexp `\[(?P<level>\w+)\]` | line_format "{{.level}}: {{.__line__}}"
   
   # Extract and format IP addresses
   {container="log-generator"} | regexp `IP (?P<ip>\d+\.\d+\.\d+\.\d+)` | line_format "User from: {{.ip}}"
   
   # Create custom log format
   {container="log-generator"} | regexp `\[(?P<level>\w+)\].*request ID (?P<id>\d+)` | line_format "Level: {{.level}}, Request: {{.id}}"
   ```

3. **Aggregation Functions:**
   ```logql
   # Count logs by level
   sum by (level) (count_over_time({container="log-generator"} | regexp `\[(?P<level>\w+)\]` [5m]))
   
   # Rate of error logs
   rate({container="log-generator"} |= "ERROR" [1m])
   
   # Count of logs per minute
   count_over_time({container="log-generator"} [1m])
   
   # Bytes processed per second
   bytes_rate({container="log-generator"} [1m])
   ```

### Exercise 4: Log Analysis Challenges
Test your LogQL skills with these real-world scenarios:

1. **Security Analysis:**
   ```logql
   # Find failed login attempts (database connection failures)
   {container="log-generator"} |= "Failed to connect"
   
   # Monitor for specific IP patterns (potential attacks)
   {container="log-generator"} | regexp `IP (?P<ip>192\.168\.\d+\.\d+)` | line_format "Internal IP: {{.ip}}"
   
   # Count unique IP addresses
   count by (ip) (count_over_time({container="log-generator"} | regexp `IP (?P<ip>\d+\.\d+\.\d+\.\d+)` [10m]))
   ```

2. **Performance Analysis:**
   ```logql
   # Find high memory usage events
   {container="log-generator"} |~ "Memory usage at [89][0-9]%" 
   
   # Extract memory percentages above 50%
   {container="log-generator"} | regexp `Memory usage at (?P<memory>[5-9][0-9])%` | line_format "High memory: {{.memory}}%"
   
   # Monitor database connection issues
   {container="log-generator"} |= "database" |= "Failed"
   ```

3. **Application Monitoring:**
   ```logql
   # Track user activity
   {container="log-generator"} |= "User logged in"
   
   # Monitor request processing
   {container="log-generator"} |= "Processing request"
   
   # Application error rate
   rate({container="log-generator"} |= "ERROR" [5m])
   ```

### Exercise 5: Time-based Log Analysis
Learn to work with time ranges and historical data:

1. **Time Range Queries:**
   ```logql
   # Last 5 minutes of logs
   {container="log-generator"}[5m]
   
   # Last hour of error logs
   {container="log-generator"} |= "ERROR" [1h]
   
   # Specific time window
   {container="log-generator"} |= "INFO" [30m]
   ```

2. **Rate and Counting Over Time:**
   ```logql
   # Error rate over last 5 minutes
   rate({container="log-generator"} |= "ERROR" [5m])
   
   # Count of warnings per minute
   count_over_time({container="log-generator"} |= "WARN" [1m])
   
   # Total bytes per second
   bytes_over_time({container="log-generator"} [1m])
   ```

3. **Trend Analysis:**
   ```logql
   # Compare error rates between time periods
   rate({container="log-generator"} |= "ERROR" [5m] offset 10m)
   
   # Sum logs by level over time
   sum by (level) (rate({container="log-generator"} | regexp `\[(?P<level>\w+)\]` [1m]))
   ```

## Common LogQL Patterns Reference

### Basic Patterns
```logql
# Stream selection
{job="containerlogs"}
{container="app-name"}

# Line filtering
|= "text"        # Contains
!= "text"        # Does not contain
|~ "regex"       # Regex match
!~ "regex"       # Regex not match

# Label extraction
| regexp `pattern`
| json
| logfmt

# Aggregations
count_over_time()
rate()
sum()
avg()
```

### Time Ranges
```logql
[5m]            # Last 5 minutes
[1h]            # Last 1 hour
[24h]           # Last 24 hours
offset 1h       # 1 hour ago
```

5. **Use Labels Efficiently:** Filter by labels before applying line filters
