# Lesson 13: Practice Lab - Monitoring with Prometheus & Grafana

## Lab Exercises

### Exercise 1: Explore Prometheus Basics
1. Access Prometheus at http://localhost:9090
2. Navigate to **Status → Targets** 
3. Verify all targets are "UP"
4. Try these basic PromQL queries in the **Graph** section:
   ```promql
   # CPU usage across all containers
   rate(container_cpu_usage_seconds_total[5m])
   
   # Memory usage by container
   container_memory_usage_bytes
   
   # Node CPU usage percentage
   100 - (avg(rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100)
   
   # Disk usage percentage
   100 - ((node_filesystem_avail_bytes / node_filesystem_size_bytes) * 100)
   ```

### Exercise 2: Advanced Prometheus Queries
Practice more complex PromQL queries to understand metrics better:

1. **Container Resource Analysis:**
   ```promql
   # Top 5 containers by CPU usage
   topk(5, rate(container_cpu_usage_seconds_total[5m]))
   
   # Memory usage as percentage of limit
   (container_memory_usage_bytes / container_spec_memory_limit_bytes) * 100
   
   # Container network received bytes rate
   rate(container_network_receive_bytes_total[5m])
   
   # Container network transmitted bytes rate
   rate(container_network_transmit_bytes_total[5m])
   ```

2. **System Performance Metrics:**
   ```promql
   # Load average
   node_load1
   
   # Memory usage percentage
   (1 - (node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes)) * 100
   
   # Disk I/O read operations rate
   rate(node_disk_reads_completed_total[5m])
   
   # Disk I/O write operations rate
   rate(node_disk_writes_completed_total[5m])
   
   # Network interface receive rate
   rate(node_network_receive_bytes_total[5m])
   
   # Network interface transmit rate
   rate(node_network_transmit_bytes_total[5m])
   ```

3. **Prometheus Internal Metrics:**
   ```promql
   # Prometheus scrape duration
   prometheus_target_scrape_duration_seconds
   
   # Number of samples ingested per second
   rate(prometheus_tsdb_symbol_table_size_bytes[5m])
   
   # Prometheus configuration reload success
   prometheus_config_last_reload_successful
   ```

### Exercise 3: Configure Grafana Dashboards
1. Access Grafana at http://localhost:3000
2. Login with `admin/admin` (change password when prompted)
3. Import these popular dashboards:
   
   **For System Monitoring:**
   - Dashboard ID: **1860** (Node Exporter Full)
   - Dashboard ID: **179** (Docker Container & Host Metrics)
   - Dashboard ID: **893** (cAdvisor)

4. **Import Process:**
   - Click **+** → **Import**
   - Enter Dashboard ID
   - Select **Prometheus** as datasource
   - Click **Import**

### Exercise 4: Create Custom Dashboard
Create a custom dashboard with multiple panels:

1. **Create New Dashboard:**
   - Click **+** → **Dashboard**
   - Add Panel

2. **Panel 1 - CPU Usage:**
   ```promql
   100 - (avg(rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100)
   ```
   - Set title: "System CPU Usage"
   - Set unit: "Percent (0-100)"
   - Add thresholds: Yellow at 60%, Red at 80%

3. **Panel 2 - Memory Usage:**
   ```promql
   (1 - (node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes)) * 100
   ```
   - Set title: "Memory Usage"
   - Set unit: "Percent (0-100)"
   - Add thresholds: Yellow at 70%, Red at 90%

4. **Panel 3 - Container Count:**
   ```promql
   count(container_last_seen)
   ```
   - Set title: "Running Containers"
   - Set visualization: "Stat"

5. **Panel 4 - Network Traffic:**
   ```promql
   rate(node_network_receive_bytes_total{device!~"lo|docker.*"}[5m])
   ```
   - Set title: "Network Receive Rate"
   - Set unit: "Bytes/sec"

### Exercise 5: Prometheus Query Challenges
Test your PromQL skills with these challenges:

1. **Challenge 1:** Find the container using the most memory
   ```promql
   # Your query here - find the answer!
   topk(1, container_memory_usage_bytes)
   ```

2. **Challenge 2:** Calculate average CPU usage over the last hour
   ```promql
   # Your query here
   avg_over_time(100 - (avg(rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100)[1h:5m])
   ```

3. **Challenge 3:** Find filesystems with less than 20% free space
   ```promql
   # Your query here
   ((node_filesystem_avail_bytes / node_filesystem_size_bytes) * 100) < 20
   ```

4. **Challenge 4:** Calculate network bandwidth utilization
   ```promql
   # Your query here
   (rate(node_network_receive_bytes_total[5m]) + rate(node_network_transmit_bytes_total[5m])) * 8
   ```

## Bonus Challenges

### Advanced PromQL Exercises

1. **Predict Disk Full Time:**
   ```promql
   predict_linear(node_filesystem_free_bytes[1h], 4 * 3600) < 0
   ```

2. **Calculate 95th Percentile Response Time:**
   ```promql
   histogram_quantile(0.95, rate(prometheus_http_request_duration_seconds_bucket[5m]))
   ```

3. **Find Containers with Restart Loops:**
   ```promql
   increase(container_start_time_seconds[1h]) > 5
   ```

4. **Calculate Error Rate:**
   ```promql
   rate(prometheus_http_requests_total{code!~"2.."}[5m]) / rate(prometheus_http_requests_total[5m]) * 100
   ```
